Overview

We are going to be making a graphic flappy bird game in python using the pygame module in python. Flappy bird is a hard game to make but easy to play. Only two choices at any given time; jump or don’t jump. The complex part of making it lies in the aesthetics and functionality of the game. The group's objective is to make a flappy bird game that looks good, allows a player to keep track of their score and is fun to play. This means things like obstacles that move, time and a score counter will need to be added.

Aesthetics

What Are aesthetics?

The definition of 'aesthetic' is 'concerned with beauty or the appreciation of beauty'. In the case it's how visually pleasing the game is. If a game has a b unhc of really bright colours and a terrible c haracter design the game will seem low quality.


I address this by...

Colors that pop out of the screen are key to graphical games. We want to try and stick to some of the original colors of the game while adding our own touches. For example the background will still be blue, and the player still yellow, but the obstacles that kill you are red now. The floor is red with a gradient to black because we want to be able to see the floor moving. If the floor moves it looks like the player is moving more than if just the columns are moving. Despite having access to all these beautiful colors one limiting aspect of the way the game is made is that we can only use a certain number of ‘pixels’. The more detailed we make it the harder it is to run the program, so rather than make a super detailed player a yellow square that moves up and down will suffice.
